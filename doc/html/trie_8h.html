◆ trie_iter_next()

void trie_iter_next ( TrieIterator **  iter_p)

Advance an iterator to the next valid (key, value) pair.

An iterator may be invalidated after instantiation only if a key is deleted which is prefixed by key_prefix and also prefixes the current iterator key.

*iter_p would be invalidated (set to NULL) if either the iterator has ended or there is no memory to proceed with the iteration.

Parameters
iter_pPointer to valid iterator or NULL
